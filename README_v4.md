# J Partner Quant System v4.0 — 업그레이드 가이드

v3.0에서 v4.0으로 업그레이드하는 방법입니다. v3.0이 이미 정상 작동 중이므로, 새 파일 세 개만 추가하면 즉시 v4.0을 사용하실 수 있습니다. 기존 파일을 덮어쓰지 않기 때문에 문제가 생기면 즉시 v3.0으로 돌아갈 수 있습니다.

## v4.0의 핵심 변화

v4.0은 정재영님이 요구하신 세 가지 핵심 사항을 모두 반영했습니다.

첫째로 4단계 스마트 스캐닝이 도입되었습니다. 코스피와 코스닥 전체 약 2,400개 종목을 무차별 분석하지 않고 광역 스캔, 신호 감지, 펀더멘털 보강, 트랙 분류의 4단계 깔때기를 통해 3분에서 5분 이내에 진짜 의미 있는 30개에서 100개 종목만 추출합니다. 시가총액 500억 미만이나 거래대금 10억 미만 같은 노이즈는 1단계에서 자동 제거되며, 거래량 폭발이나 외국인 매수 전환 같은 신호가 있는 종목만 정밀 분석으로 넘어갑니다.

둘째로 보수형과 공격형 듀얼 모드가 추가되었습니다. 같은 데이터로 두 가지 시각의 결과가 동시에 화면에 좌우로 표시됩니다. 보수형은 펀더멘털과 손익비를 엄격하게 검증한 안전 우선 종목 위주이며, 공격형은 일부 필터를 완화하여 RSI 과열 구간이나 적자 기업이라도 모멘텀이 강한 종목을 함께 노출합니다. 정재영님이 그날 시장 분위기와 컨디션에 따라 어느 쪽을 따를지 즉시 결정하실 수 있습니다.

셋째로 자동 캐시와 백그라운드 실행이 가능해졌습니다. 한 번 스캔한 결과는 JSON 파일로 저장되어 다음에 접속할 때 즉시 표시되며, 스케줄러를 통해 매일 자동으로 갱신할 수 있습니다.

## 추가할 새 파일 3개

기존 폴더(`Desktop\j_quant_system`)에 다음 세 개의 파일을 추가하시면 됩니다.

첫 번째는 `j_engine_v4.py`로 보수형과 공격형 듀얼 모드 J 엔진입니다. 기존 j_engine.py와 별도로 존재하며 충돌하지 않습니다. 두 번째는 `data_collector_v4.py`로 4단계 스마트 스캐닝과 pykrx 거래일 안정화가 포함된 신규 데이터 수집기입니다. 세 번째는 `app_v4.py`로 보수형과 공격형을 좌우로 동시 표시하는 새 Streamlit 웹앱입니다. 네 번째는 `scheduler_v4.py`로 매일 자동 실행과 이메일 발송, JSON 캐시 저장을 통합한 자동 스케줄러입니다.

기존 파일들(j_engine.py, data_collector.py, app.py, scheduler.py, config.py, requirements.txt)은 그대로 두시기 바랍니다. v4.0은 별도 파일로 작동하므로 기존 v3.0과 공존합니다.

## 실행 방법

기존 Streamlit 서버가 실행 중이라면 명령 프롬프트에서 Ctrl+C로 먼저 종료하시기 바랍니다. 그 다음 새 v4.0 앱을 실행합니다.

```
python -m streamlit run app_v4.py
```

브라우저가 열리면 새로운 v4.0 화면이 표시됩니다. 가장 큰 차이는 화면 상단에 "스캐닝 방식 선택"이라는 라디오 버튼이 다섯 가지로 늘어난 것입니다. 첫째 옵션인 전체 스마트 스캔을 선택하시면 3에서 5분간 코스피와 코스닥 전체를 자동 분석합니다. 둘째 옵션인 관심 종목 빠른 수집은 1분 이내에 20개 관심 종목만 빠르게 가져옵니다. 셋째 옵션인 샘플 데이터는 즉시 시연용으로 결과를 보여줍니다. 넷째 옵션인 JSON 직접 입력은 기존 v3.0과 동일합니다. 다섯째 옵션인 마지막 스캔 결과 불러오기는 캐시 파일을 즉시 표시합니다.

처음 사용 시에는 셋째 옵션인 샘플 데이터로 보수형과 공격형 화면이 좌우로 어떻게 나오는지 먼저 확인하시고, 그 다음 둘째 옵션인 관심 종목 빠른 수집을 시도하시는 것을 권장합니다. 안정화가 확인되면 첫째 옵션인 전체 스마트 스캔을 본격적으로 사용하시면 됩니다.

## 자동 스케줄러 설정

매일 평일 오후 4시 30분에 자동으로 전체 스캔이 실행되도록 설정하려면 Windows 작업 스케줄러를 사용합니다. 시작 메뉴에서 작업 스케줄러를 검색해 실행하시고 기본 작업 만들기를 클릭하시기 바랍니다. 이름에 J Quant v4.0 Daily Scan이라고 입력하시고 트리거는 매일, 시작 시간은 오후 4시 30분으로 설정하시면 됩니다. 동작은 프로그램 시작을 선택하시고 프로그램에는 python을 입력하시며, 인수에는 scheduler_v4.py를 입력하시고, 시작 위치에는 시스템 폴더 경로인 C:\Users\정재영\Desktop\j_quant_system을 입력하시기 바랍니다.

설정 완료 후에는 매일 평일 오후 4시 30분에 자동으로 전체 시장 스캔이 실행되고, 결과가 JSON 캐시로 저장되며, config.py에 Gmail 정보가 입력되어 있다면 이메일로도 리포트가 발송됩니다. PC가 켜져 있어야만 작동한다는 점만 주의하시면 됩니다.

## 모바일에서 사용하기

PC와 같은 Wi-Fi에 연결된 휴대폰에서 모바일 브라우저를 열고 PC의 Network URL인 192.168.35.159:8501에 접속하시면 동일한 v4.0 화면이 모바일에서도 표시됩니다. 화면이 모바일 크기에 자동으로 조정되며, 좌우 분할된 보수형과 공격형 화면은 위아래로 자동 재배치되어 가독성을 유지합니다.

더 편리하게 사용하시려면 마지막 단계인 Streamlit Cloud 클라우드 배포를 진행하시면 됩니다. PC가 꺼져 있어도 24시간 모바일에서 접속 가능한 진정한 옵션 C가 완성됩니다. 클라우드 배포는 v4.0이 로컬에서 안정적으로 작동함을 확인한 후 진행하시는 것이 안전합니다.

## 향후 업데이트 예정 사항

차후 업데이트에서 다음 기능들이 단계적으로 추가됩니다. 외국인과 기관 쌍끌이 시그널 자동 감지, 공매도 잔고 추적, 신고가와 박스권 돌파 패턴 자동 감지, 분기 실적 시즌 자동 활용, 같은 섹터 동시 상승 감지를 통한 신규 테마 자동 발굴, 매수 매도 패턴 학습을 통한 개인화, 보유 종목과의 상관관계 분석을 통한 리스크 분산, 시장 국면 자동 적응, 백그라운드 자동 실행을 통한 모바일 접속 시 즉시 표시, 그리고 네이버 금융 뉴스 자동 수집과 키워드 기반 호재 악재 분류입니다. 모든 기능은 v4.0이 안정화된 후 정재영님의 피드백을 반영하여 우선순위에 따라 추가됩니다.

## 주의 사항

전체 스마트 스캔의 첫 실행은 pykrx가 KRX 서버에서 약 2,400개 종목의 데이터를 가져와야 하므로 3분에서 5분 정도 걸립니다. 중간에 일부 종목 수집이 실패할 수 있지만 시스템이 자동으로 건너뛰고 다음 종목으로 넘어가므로 전체 작업은 중단되지 않습니다. 인터넷 연결이 안정적인 환경에서 실행하시는 것이 좋습니다.

전체 스캔 결과 통과 종목이 너무 적게 나오면 시스템 시작 시 광역 스캔의 시가총액 최소값이나 거래대금 최소값을 낮추실 수 있습니다. 반대로 너무 많이 나오면 수치를 높여서 더 엄격한 필터링을 적용하실 수 있습니다.

Python 3.14의 pykrx 호환성 문제로 일부 함수가 정상 작동하지 않을 수 있습니다. 그런 경우 setuptools 70.0.0이 설치되어 있는지 확인하시고, 필요하면 Python 3.12로 다운그레이드를 고려해 보실 수 있습니다. 다만 우선 v4.0을 실행해 보시고 어떤 오류가 발생하는지 확인한 후 결정하시는 것이 좋습니다.

본 시스템은 의사결정 보조 도구이며 모든 매매 결정 전 HTS 실시간 가격을 직접 확인하시는 것이 J 파트너 운용 지침 V1.0의 핵심 원칙입니다. 시스템이 추천한 종목이라도 반드시 본인 판단을 거쳐 매수하시기 바랍니다.
